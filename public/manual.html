1. Manager Daily Workflow

  1. Check team KPI trend and risks in Dashboard.
  2. Input KPI achievements and competency assessments in Assessment & KPI.
  3. Manage team KPI definitions and monthly targets in Settings > KPI Management.
  4. Review probation and follow-up PIP in Records > Probation & PIP.

2. Add/Edit KPI Definition (Manager)

  1. Open Settings > KPI Management > KPI Definition.
  2. Fill KPI name, category/position, target, effective month, and unit from dropdown.
  3. Click Save. If governance is enabled, request goes to pending approval.
  4. Track progress in Pending KPI Approvals and KPI Version History.

3. Monthly Target Config

  1. In Employee Targets Config, select target month and employee.
  2. Enter monthly override values per KPI for that employee.
  3. Leave blank to use default KPI target.
  4. Save and re-check values in KPI history/records.

4. KPI Record Flow

  1. Go to Assessment & KPI, choose employee, click Input KPI Achievement.
  2. Pick period, KPI metric, and value.
  3. If unit is %, fill Attained + Total to auto-calculate.
  4. Validate saved data in Records > KPI Records.

5. Probation & PIP

  1. Use Review Form to fill Managing score and monthly qualitative text.
  2. Work score comes from KPI records in probation date window.
  3. Attitude score comes from attendance deductions entered by HR/Super Admin.
  4. Set decision and export report for signature process.

6. Troubleshooting (Manager)

  • If KPI save fails, verify you are editing team position only and fields are complete.
  • If KPI update is pending, wait for HR/Super Admin approval.
  • If KPI card count looks low, re-check active period filter.

1. Login & Account

  • Use registered email and password on login page.
  • Use Forgot Password to request reset link.
  • Use header Password action to update password.

2. Core Workflow (HR/Admin)

  1. Check KPI summary and analytics in Dashboard.
  2. Maintain employee and assessment data in Assessment & KPI.
  3. Review outputs in Records.
  4. Maintain app/user/configuration in Settings.

3. KPI Governance & Config

  1. Use KPI Governance toggle to require HR approval for manager KPI changes.
  2. Review and approve pending definition/target requests.
  3. Set probation scoring policy in Settings > General (weights, threshold, attendance rules).
  4. Use version history to audit KPI definition and target changes.

4. KPI Records & Reporting

  • KPI records store KPI snapshot (name/unit/category/target) at submission time.
  • Use filters (Department, Manager, Period) before exporting.
  • Use Records exports (CSV/Excel/PDF) for operational and approval workflows.

5. Probation Process

  1. Generate probation drafts for employees nearing probation end.
  2. Manager fills monthly qualitative review; HR inputs attendance deductions.
  3. Review final score, pass threshold, and decision (Pass/Extend/Fail).
  4. Export signed report package for approval and compensation process.
Probation windows are date-to-date from join date, not fixed calendar month.

6. Attendance Deduction Rules Setup

  1. Open Settings > General.
  2. Find Probation Attendance Deduction Rules.
  3. Use Reset Template to restore the default structure or Format JSON to clean up spacing.
  4. monthly_cap is the maximum deduction allowed in one month.
  5. Each event inside events must have a user-friendly label.
  6. Use mode: "per_qty" for simple rules like 1 point per incident, with per_qty and max_points.
  7. Use mode: "tiered" for threshold-based rules, with a tiers list of min_qty and points.
Example: { "monthly_cap": 20, "events": { "late_in": { "label": "Late Clock In", "mode": "tiered", "tiers": [ { "min_qty": 3, "points": 1 } ] } } }

7. Troubleshooting

  • Missing menu/action: check role and filter scope first.
  • KPI request not active: verify pending approval status.
  • Auth error email rate limit exceeded: wait and retry later.

1. Workflow Harian Manager

  1. Cek tren KPI tim dan risiko di Dashboard.
  2. Input KPI achievement dan assessment kompetensi di Assessment & KPI.
  3. Kelola definisi KPI dan target bulanan tim di Settings > KPI Management.
  4. Review probation dan tindak lanjut PIP di Records > Probation & PIP.

2. Tambah/Edit Definisi KPI (Manager)

  1. Buka Settings > KPI Management > KPI Definition.
  2. Isi nama KPI, kategori/posisi, target, effective month, dan unit dari dropdown.
  3. Klik Save. Jika governance aktif, perubahan menjadi pending approval.
  4. Pantau status di Pending KPI Approvals dan KPI Version History.

3. Konfigurasi Target Bulanan

  1. Di Employee Targets Config, pilih bulan target dan karyawan.
  2. Isi override target bulanan per KPI.
  3. Kosongkan nilai jika ingin pakai target default.
  4. Simpan lalu validasi di KPI history/records.

4. Alur KPI Record

  1. Masuk ke Assessment & KPI, pilih karyawan, klik Input KPI Achievement.
  2. Pilih period, metric KPI, lalu isi nilai.
  3. Jika unit %, isi Attained + Total agar auto-calculate.
  4. Validasi hasil di Records > KPI Records.

5. Probation & PIP

  1. Isi Managing score dan qualitative text per bulan pada form review.
  2. Work score diambil dari KPI records dalam window probation.
  3. Attitude score berasal dari attendance deduction oleh HR/Super Admin.
  4. Tentukan keputusan dan export report untuk proses tanda tangan.

6. Troubleshooting (Manager)

  • Jika save KPI gagal, pastikan KPI pada posisi tim dan field wajib terisi.
  • Jika perubahan KPI pending, tunggu approval HR/Super Admin.
  • Jika jumlah record KPI kecil, cek filter period aktif.

1. Login & Akun

  • Masuk dengan email dan password terdaftar.
  • Gunakan Forgot Password untuk reset password.
  • Gunakan aksi Password di header untuk ganti password.

2. Workflow Utama (HR/Admin)

  1. Cek ringkasan KPI dan analitik di Dashboard.
  2. Kelola data karyawan dan assessment di Assessment & KPI.
  3. Review hasil final di Records.
  4. Kelola user, setting, dan konfigurasi di Settings.

3. KPI Governance & Konfigurasi

  1. Gunakan toggle KPI Governance untuk mewajibkan approval HR pada perubahan KPI oleh manager.
  2. Review dan approve request definisi/target KPI yang pending.
  3. Atur kebijakan probation di Settings > General (bobot, threshold, aturan attendance).
  4. Gunakan version history untuk audit perubahan KPI definition dan target.

4. KPI Records & Reporting

  • KPI record menyimpan snapshot KPI (nama/unit/kategori/target) saat submit.
  • Gunakan filter Department, Manager, dan Period sebelum export report.
  • Gunakan export Records (CSV/Excel/PDF) untuk kebutuhan operasional dan approval.

5. Proses Probation

  1. Generate draft probation untuk karyawan yang mendekati akhir probation.
  2. Manager isi review kualitatif bulanan, HR isi attendance deduction.
  3. Review final score, pass threshold, dan keputusan (Pass/Extend/Fail).
  4. Export dokumen report untuk proses tanda tangan dan penyesuaian kompensasi.
Window probation dihitung date-to-date dari join date, bukan bulan kalender tetap.

6. Setup Aturan Attendance Deduction

  1. Buka Settings > General.
  2. Cari field Probation Attendance Deduction Rules.
  3. Gunakan Reset Template untuk mengembalikan format default atau Format JSON untuk merapikan isi JSON.
  4. monthly_cap adalah batas maksimal deduction dalam satu bulan.
  5. Setiap event di dalam events wajib punya label yang mudah dibaca user.
  6. Gunakan mode: "per_qty" untuk aturan sederhana, misalnya 1 poin per kejadian, dengan field per_qty dan max_points.
  7. Gunakan mode: "tiered" untuk aturan bertingkat, dengan daftar tiers berisi min_qty dan points.
Contoh: { "monthly_cap": 20, "events": { "late_in": { "label": "Late Clock In", "mode": "tiered", "tiers": [ { "min_qty": 3, "points": 1 } ] } } }

7. Troubleshooting

  • Jika menu/aksi tidak muncul, cek role dan scope filter.
  • Jika perubahan KPI belum aktif, cek status pending approval.
  • Error auth email rate limit exceeded: tunggu lalu coba lagi.
